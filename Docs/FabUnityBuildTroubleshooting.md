# Fab Unity Build Troubleshooting

Last updated: 2026-05-18

## Incident: Anonymous Namespace Symbols Collide In Fab

Fab compiled generated unity blobs such as:

- `Module.ComposableCameraSystem.10.cpp`
- `Module.ComposableCameraSystem.11.cpp`
- `Module.ComposableCameraSystemEditor.3.cpp`

Those files are generated by UnrealBuildTool. They are not source files to edit. When Fab builds with unity enabled, several `.cpp` files are included into the same translation unit, so identical helpers in anonymous namespaces can collide.

The failing symbols were:

- `AddVectorOutputSlot`
- `DrawCanvasRect`
- `StartupCallbackHandle`
- `ResolveLiveSequencers`

Nearby symbols with the same risk were also fixed:

- `ResolveLSComponent`
- `ResolvePCM`

Fix pattern: give anonymous-namespace helpers and variables file-specific names, then update only the call sites in that same `.cpp` file. Do not disable unity build as the marketplace fix.

## Critical Path Guardrail

Before editing or verifying any plugin issue, confirm the exact source root.

For this incident, the first edit was made in the wrong copy:

`C:\Users\Sulley\Documents\ComposableCameraSystem\HostProject\Plugins\ComposableCameraSystem`

The intended Fab/UE5.6 plugin source root was:

`C:\Users\Sulley\Documents\Unreal Projects\UE5_6\Plugins\ComposableCameraSystem`

Rule: a user-provided absolute path beats recursive search results. Never edit the first matching plugin copy just because it was found first.

## Source Root Checklist

Before writing files, verify all of these:

- Plugin root is the user's intended root, usually `<Project>/Plugins/<PluginName>`.
- `.uproject` matches the target engine version.
- Build command points at the same `.uproject` that will be packaged/submitted.
- Ignore `Intermediate`, `Binaries`, `Saved`, `Cooked`, and `Temp` as source locations.
- Treat `Module.*.cpp` paths as unity-build diagnostics, not hand-edit targets.

For this UE5.6 plugin, the expected pair is:

- Plugin: `C:\Users\Sulley\Documents\Unreal Projects\UE5_6\Plugins\ComposableCameraSystem`
- Project: `C:\Users\Sulley\Documents\Unreal Projects\UE5_6\UE5_6.uproject`

## Local Reproduction Command

Use forced unity and disable adaptive unity. Adaptive unity can exclude recently edited files and hide the same failure Fab catches.

```powershell
& "C:\Program Files\Epic Games\UE_5.6\Engine\Build\BatchFiles\Build.bat" `
  UnrealEditor Win64 Development `
  -Project="C:\Users\Sulley\Documents\Unreal Projects\UE5_6\UE5_6.uproject" `
  -ForceUnity `
  -DisableAdaptiveUnity `
  -WaitMutex `
  -NoHotReloadFromIDE
```

Expected evidence after the fix: the build succeeds while compiling the generated unity blobs that previously failed, including `Module.ComposableCameraSystem.10.cpp`, `Module.ComposableCameraSystem.11.cpp`, and `Module.ComposableCameraSystemEditor.3.cpp`.
